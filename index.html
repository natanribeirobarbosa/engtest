Simulado – Aula 6: Análise de Requisitos

1. O que é Engenharia de Requisitos?

Uma linguagem de programação
Processo de descobrir e analisar requisitos
Uma técnica de teste

2. Qual das opções é um requisito funcional?

Tempo de resposta
Gerar nota fiscal eletrônica
Confiabilidade

3. Requisitos não funcionais tratam de:

O que o sistema deve fazer
Qualidades do sistema
Erros do usuário

4. Um requisito funcional oculto é aquele que:

É realizado internamente sem o usuário perceber
Está visível na tela do sistema
Não é documentado

5. O que significa um requisito mensurável?

Pode ser apagado
Pode ser validado com uma métrica
Não pode ser alterado

6. Um bom requisito deve ser:

Vago
Claro e objetivo
Subjetivo

7. Qual é a função de um estudo de viabilidade?

Reduzir custos
Investigar se o projeto é possível
Testar funcionalidades

8. Sustentabilidade em requisitos significa:

Uso de energia
Benefício residual prolongado
Obsolescência

9. Um exemplo de requisito não funcional é:

Cadastrar usuário
Tempo de resposta de até 2s
Emitir boleto

10. O que representa um requisito relevante?

Requisito repetido
Necessidade real do projeto
Requisito que pode ser ignorado
