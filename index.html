Resumo - Aula 6: Análise de Requisitos

Engenharia de Requisitos

É o processo relacionado à descoberta e análise dos requisitos de um sistema.

O que são Requisitos?

São descrições das funções (o que o sistema faz) e restrições (como ele deve ser) de um sistema.

Importância dos Requisitos

Os requisitos definem o desempenho, funcionalidades e aspectos de desenvolvimento do sistema. Eles devem ser:

  • Claros: cada requisito deve ser direto e sem ambiguidades.
  • Relevantes: devem refletir necessidades reais.
  • Mensuráveis: devem ser verificáveis com métricas.
  • Alcançáveis: devem ser viáveis técnica, econômica e temporalmente.

Classificação dos Requisitos

Requisitos Funcionais: descrevem ações que o sistema deve executar. Ex.: emitir notas fiscais.

Requisitos Não Funcionais: descrevem qualidades do sistema (ex.: confiabilidade, segurança).

Tabelas de Requisitos

Funcionais: código, nome, descrição, categoria (evidente/oculto).

Não Funcionais: nome, restrição, tipo, obrigatoriedade.

Estudo de Viabilidade

Deve ser planejado, ouvir várias opiniões, prever riscos e ser feito com profissionalismo.

Sustentabilidade

Continuidade dos aspectos econômicos, sociais e ambientais após o desenvolvimento.

Workshop

Atividade em grupo para levantar e apresentar requisitos de sistemas diversos.
