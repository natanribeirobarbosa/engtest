Aula 6 - Análise de Requisitos

Engenharia de Requisitos

É o processo relacionado à descoberta e análise dos requisitos de um sistema.

O que são Requisitos?

São descrições das funções (o que o sistema faz) e restrições (como ele deve ser) de um sistema.

Importância dos Requisitos

Os requisitos definem o desempenho, funcionalidades e aspectos de desenvolvimento do sistema. Eles devem ser:

- Claros: cada requisito deve ser direto e sem ambiguidades.

- Relevantes: devem refletir necessidades reais.

- Mensuráveis: devem ser verificáveis com métricas.

- Alcançáveis: devem ser viáveis técnica, econômica e temporalmente.

Classificação dos Requisitos

Requisitos Funcionais: descrevem ações que o sistema deve executar.

Exemplos:

- O sistema deve emitir notas fiscais eletrônicas.

- O sistema deve enviar e-mails com as notas fiscais emitidas.

Requisitos Não Funcionais: descrevem qualidades do sistema (confiabilidade, tempo de resposta, compatibilidade, segurança, etc.).

Tabelas de Requisitos

Requisitos Funcionais: devem ter código, nome, descrição e categoria (evidente ou oculto).

Requisitos Não Funcionais: devem conter nome, restrição, tipo e obrigatoriedade.

Estudo de Viabilidade

É um investimento em conhecimento. Deve ser feito com planejamento, ouvindo diferentes pontos de vista, considerando riscos e de forma profissional.

Sustentabilidade

Busca a continuidade dos aspectos econômicos, sociais e ambientais de um sistema após seu desenvolvimento.

Workshop

Atividade em grupo para levantar e tabular requisitos de sistemas diversos (drogaria, hospital, faculdade, etc.), com apresentação posterior.
